Linux 生产服务器被kdevtmpfsi 挖矿程序入侵导致CPU占用100%

前言

今日下午，准备配置生产服务器上的nginx以及ActiveMQ等第三方工具时，

首先习惯性先打free -h查看内存占用情况，一看如下图，还好，此服务器跑了ActiveMQ，Nginx，mongodb，redis，Nacos等

但当打top，查看CPU及负载均衡的使用情况时，一看，吓一跳，因为这台服务器并还未真正用于生产，仅预备启动了一些服务

主要的罪魁祸首为名为kdevtmpfsi的程序

百度/google一波之后，发现竟为挖矿程序。。

导致原因为redis-server的bind 地址为0.0.0.0 并且redis-server 的requirepass并没有设置，导致被程序入侵

尝试过直接kill掉此进程

但几分钟之后又卷土重来。。。

后看到此挖矿程序有守护进程，会自动重新此程序

·```shell
[root@iZuf6grnhznt1ut5yl83wzZ tmp]# ps -ef|grep kinsing
root 8867 30307 0 14:38 pts/0 00:00:00 grep –color=auto kinsing
root 14078 1 0 Dec25 ? 00:01:50 /var/tmp/kinsing

多了一段时间，再次top，load average已经达到了正常的范围，且没有占用CPU异常的程序了

至此，kdevtmpfsi 挖矿程序占用服务器CPU的问题到此解决(吓得我…..)

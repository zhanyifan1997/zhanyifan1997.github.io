SpringCloud Gateway 分布式限流配置

准备工作

一、创建两个微服务

1.1 Gateway project

1.1.1 导入依赖 主要依赖为 spring-cloud-starter-gateway和 spring-boot-starter-data-redis-reactive 其他为整合Nacos作为分布式配置中心的其他依赖
1.1.2 进行gateway 微服务配置

1.2 limit-micro project

1.2.1 导入依赖，为单独的一个SpringBoot项目，但此SpringBoot项目注册进入了配置中心，Gateway通过微服务名称可以找到对应的所在ip
1.2.2 进行limit-microservice 微服务配置
1.2.3 编写一个简单的controller

1.3 启动项目，采用jmeter进行测试

可以看到模拟20个线程进行访问，因为设置了5个令牌的数量限制，只有5-6个请求能通过，其他请求均返回429错误码

踩坑点

因为采用了SpringCloud gateway，能够通过微服务名称找到对应的微服务，进行访问如果直接访问对应的provider微服务提供者，那么限流是能够生效的，但是，如果采用feign/ribbon进行客户端的访问，即gateway 调用feign，feign调用对应的provider那对应的限流是无效的

解决办法为：

配置SpringCloud gateway时，将feign配置为第一项，并且调用feign时，不写feign对应微服务的名称，而是直接通过gateway的根路径进行调用（默认找配置的第一个router）
访问时直接通过 http://gatewayip:gateway-port/feign调用provider的地址

SpringCloud gateway 限流到此配置完毕
